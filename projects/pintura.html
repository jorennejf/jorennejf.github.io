Pintura: Geographical Digital Graffiti App

June 2017


This is a low-fidelity video we created during one of our class sessions in which we showcase what our app is.


About

Pintura is a web app in which users can post and view digital graffiti based on their current location. The goal of our app is to allow users to express their creativity while creating a sense of community and exploration. Traditional graffiti defaces public property, so by moving it to a digital platform, it allows artists to continue to express themselves through lawful methods.

This project was a part of COGS 121: HCI Programming Studio, where we worked in teams of 4 to create geographical and map-related applications. The other members of my team include Connie Guan, Tony Salim, and Chang Gwoun.

    Tools

  • JavaScript
  • HTML
  • CSS

    My Roles

  • Prototyping
  • Front-End Development
  • Graphics

Main Features

  • Users can virtually tag locations, and these places are marked on the map
  • Interactive map allows the user to browse for nearby graffiti
  • Upvoting system in which graffiti with more upvotes can be seen from longer distances than less popular graffiti

Design Process

    Ideate

  1. We all brainstormed different ideas in which the problem can be solved with a map or GPS application. To do this, we created storyboards of different scenarios and presented them to other teams in the class.

    Prototype

  2. In class, we had 2 paper prototyping sessions. We gave and received feedback for both, and we used this feedback to modify our UI and app functionality. Iterative design really helped guide us in determining what features we wanted to include.

    For the first session, my teammates and I drew prototypes of all the pages so that we can get a broader range of UIs and choose what we liked from them.






    This prototype shows the home screen of our app, which is a map with the user's current location and locations of graffiti.


    This prototype is another version of the viewing page. It is meant to replicate a wall of graffiti. Users can scroll right to view other graffiti.


    This prototype is the drawing page that is accessed when the user clicks "Draw" on the home screen. The majority of the page is dedicated to drawing the user's graffiti, with the available tools on the top.
    For the second round of prototyping, each of us focused on one feature of the app. My task was to create prototypes for user profiles.
    This prototype displays the UI of the profiles in mobile view. This prototype displays the profiles again but in desktop view. It displays the exact same content as in mobile but with a different layout.

    3. Develop

  3. Our first priority was developing the core map interactions, as this was the main functionality of the app. We did this by using Leaflet to provide location-based data. At the same time, we also began development on the drawing functionality, using Sketch.js. Once these features worked properly, we began work on UI enhancements and branding, and we continued to add more features based on feedback from our peers.

Conclusion

I think I would say this is one of my proudest projects by far. I'm really happy with its uniqueness, and we got a generally positive reaction from the rest of the class. We presented our final product in the form of a silent demo video while my team each took turns narrating in front of the class.
